# Backend — Архитектура и ключевые решения

## Стек
- **FastAPI**
- **SQLAlchemy 2.0**
- **PostgreSQL**
- **Alembic**
- **python-docx + matplotlib** - генерация отчётов
- **JWT (PyJWT) + argon2** - аутентификация и хеширование паролей
- **GigaChat API** - генерация текстовых выводов по результатам теста

## Архитектура

Приложение построено по **трёхслойной архитектуре** внутри монолита:

```
HTTP-запрос
    ↓
Роутер (routers/)        — принимает запрос, валидирует через схемы, вызывает сервис
    ↓
Сервис (services/)       — бизнес-логика, не знает об HTTP
    ↓
Модель (models/)         — SQLAlchemy ORM, описывает таблицы БД
    ↓
PostgreSQL
```

Каждый слой отвечает только за своё — роутеры не содержат бизнес-логики, сервисы не знают про HTTP-статусы.

## Модели и схема БД

### Основные таблицы

| Таблица            | Описание                           |
| ------------------ | ---------------------------------- |
| `users`            | Психологи и администраторы системы |
| `tests`            | Тесты созданные психологами        |
| `sections`         | Разделы теста                      |
| `questions`        | Вопросы теста                      |
| `formulas`         | Формулы для подсчёта метрик        |
| `question_metrics` | M2M связь вопросов и формул        |
| `sessions`         | Сессии прохождения теста гостями   |
| `answers`          | Ответы гостей на вопросы           |

### Ключевые решения по моделям

**JSONB для гибких данных.** Поля `config` (вопросы), `client_extra` (доп. поля гостя), `metrics` (результаты), `report_template` (шаблон отчёта) хранятся как JSONB. Это позволяет не менять схему БД при добавлении новых типов вопросов или полей отчёта.

**Вес ответа сохраняется в момент ответа.** Поле `answers.weight` фиксирует вес в момент прохождения. Если психолог изменит веса вопросов — старые результаты не пересчитываются, история остаётся неизменной.

## Сервисный слой

### `services/test_io.py`

Центральный модуль работы с тестами. Содержит:

- **`import_test`** — разбивает большой JSON теста по таблицам БД. Генерирует новые UUID для всех объектов (тест, секции, вопросы, формулы), обновляет `next_question_id` в опциях и ссылки в `report_template` на новые UUID.
- **`export_test`** — собирает полный JSON теста из БД обратно в формат импорта (включая веса).
- **`get_test_for_guest`** — отдаёт тест гостю без весов и `score_ranges`. Все вопросы включая скрытые — фронт управляет видимостью через флаг `hidden_by_default`.
- **`save_session_answers`** — принимает ответы от фронта, вычисляет вес каждого ответа через `resolve_answer_weight`, сохраняет в таблицу `answers`.
- **`calculate_and_save_metrics`** — после сохранения ответов считает все метрики по формулам (`sum`, `avg`, `min`, `max`, `percent`), находит интерпретацию по диапазонам, сохраняет в `session.metrics`.

### `services/report_service.py`

Генерация docx-отчётов на лету без сохранения на диск:

- Два отчёта: для гостя (`report_template.client`) и для психолога (`report_template.psychologist`).
- Шаблон отчёта состоит из блоков четырёх типов: `text`, `metric`, `chart`, `answers`.
- Графики (`bar`, `radar`) рисуются через matplotlib в `BytesIO` и вставляются в docx как изображения.
- Готовый документ отдаётся через `StreamingResponse` — файл никогда не попадает на диск сервера.

### `services/gigachat.py`

Интеграция с GigaChat API для генерации текстового вывода по результатам прохождения теста.

## Аутентификация

Двухтокенная схема:

- **Access token** — короткоживущий JWT, передаётся в заголовке `Authorization: Bearer`.
- **Refresh token** — долгоживущий, хранится в БД в таблице `refresh_tokens` в виде SHA-256 хеша. Используется для получения нового access token.

Роли: `admin`, `psychologist`. Доступ к эндпоинтам контролируется через FastAPI dependencies (`get_current_psychologist`, `get_current_active_psychologist`).

Проверка активности аккаунта через `user.has_active_access` — учитывает флаг `is_blocked` и поле `access_until`.

## Флоу прохождения теста

```
1. GET  /guest/{access_link}              — получить тест (без весов)
2. POST /guest/{access_link}/start        — создать сессию, получить session_id
3. POST /guest/sessions/{id}/answers      — отправить ответы
                                            → автоматически считаются метрики
4. GET  /guest/sessions/{id}/report       — скачать docx-отчёт
```

---

## Импорт и экспорт тестов

Психолог работает с тестом через единый JSON-формат. При импорте все UUID генерируются заново — каждый импорт создаёт независимую копию теста. Это позволяет делиться шаблонами тестов между психологами без конфликтов.

`next_question_id` в опциях вопросов (ветвление) автоматически обновляется на новые UUID при импорте.

## Хранение медиафайлов

Изображения сохраняются локально в директорию `media/` через утилиту `core/image.py`. Поддерживаются форматы JPEG, PNG, WebP. Максимальный размер файла — 2 МБ. Файлу присваивается UUID-имя для избежания коллизий.
